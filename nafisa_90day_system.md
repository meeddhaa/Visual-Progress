# NAFISA'S 90-DAY SYSTEM TO HIGH-PAYING REMOTE ROLE

**Goal:** Land a $60k+ USD remote Technical PM / Product Engineer / AI Product role in 90 days.

**Current Reality:** Strong resume, no formal PM title, needs income ASAP, wants to code + think product.

**Strategy:** Position yourself as a Technical Product Builder through applications + public building + strategic networking.

---

## DAILY ROUTINE (Monday-Friday)

### Morning Block (9:00 AM - 12:00 PM) — 3 HOURS

**9:00 - 9:15 AM: Dashboard Check & Day Planning**
- Open your dashboard (we're building this)
- Review yesterday's progress
- Set today's 3 priorities
- Check application pipeline status

**9:15 - 11:00 AM: Job Applications & Outreach**
- **Target: 2 applications per day (10/week minimum)**
- Search for roles on: RemoteOK, AngelList, LinkedIn, YC Jobs, AI company career pages
- Customize cover letter per role (use template, personalize 3 sentences)
- Track every application in dashboard
- Do 1 LinkedIn/Twitter engagement (comment on a post in your target space)

**11:00 AM - 12:00 PM: Interview Prep OR Strategic Networking**
- If no interviews scheduled: reach out to 1-2 people at companies you applied to (not recruiters — PMs, engineers, founders)
- If interviews coming: prep for specific company and role
- Watch PM interview videos, practice product thinking questions

**Daily Application Rule:** Don't end the morning until 2 applications are submitted and tracked.

---

### Afternoon Block (2:00 PM - 5:00 PM) — 3 HOURS

**2:00 - 4:30 PM: BUILD SOMETHING**

This is non-negotiable. You need to ship things publicly while job hunting. Pick ONE for the month:

**Option A: Build a small AI tool (Week 1-2)**
- Example: Bengali-English AI writing assistant, AI habit coach for students, AI email writer
- Ship it to GitHub, post about it on LinkedIn
- This becomes portfolio + proof you ship

**Option B: Write technical content (Week 3-4)**
- Document how you built Lazy Daisy, or lessons from RLHF work
- Post on Medium or LinkedIn articles
- This builds thought leadership

**Option C: Improve existing projects (ongoing)**
- Polish Lazy Daisy for Play Store launch
- Add features to Raccoon Mode
- Document product decisions made

**Why this matters:** Every week you're building publicly, you're proving you ship. Hiring managers check your GitHub and LinkedIn. Empty = red flag. Active = credible.

**4:30 - 5:00 PM: Document what you built**
- Write a short post about today's progress
- Update GitHub README
- Screenshot/demo if relevant
- Queue it for posting later

---

### Evening Block (9:00 PM - 10:00 PM) — 1 HOUR

**9:00 - 9:30 PM: Dashboard Update**
- Log all applications sent today
- Update interview pipeline
- Mark tasks complete
- Track any responses received

**9:30 - 10:00 PM: Learning (optional but recommended)**
- Read PM content: Lenny's Newsletter, First Round Review, product Twitter
- Watch AI product content: learn what's happening in the space
- This gives you things to talk about in interviews

**Application Template:**
You need a base cover letter that you customize per role. Structure:

```
Hi [Name],

I'm reaching out about [Role] at [Company]. I've been building AI-integrated mobile products for the past 2 years — most recently [specific project] which [specific outcome].

What caught my attention about [Company] is [specific thing about their product or mission]. I've been thinking a lot about [relevant product problem they're solving], and I'd love to explore how I could contribute to [specific area].

I'm a technical product builder — I code (Flutter, Python, AI APIs) but I think product-first. My background includes:
- Building and shipping [X] which [outcome]
- RLHF evaluation work at RWS Group (production scale LLM training)
- [Specific relevant achievement]

Would you be open to a conversation about how I might be able to help with [their specific challenge]?

Resume attached.

Best,
Nafisa
```

Customize the bracketed parts. Keep it under 150 words. Send it.
